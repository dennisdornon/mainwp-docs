\doxysubsubsection{\texorpdfstring{enableContinuousBuffer()}{enableContinuousBuffer()}}
{\footnotesize\ttfamily \label{classphpseclib3_1_1_crypt_1_1_triple_d_e_s_a15c859d30be71e9028ff6265b2b69388} 
phpseclib3\textbackslash{}\+Crypt\textbackslash{}\+Triple\+DES\+::enable\+Continuous\+Buffer (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})}

Treat consecutive "{}packets"{} as if they are a continuous buffer.

Say you have a 16-\/byte plaintext \$plaintext. Using the default behavior, the two following code snippets will yield different outputs\+:

{\ttfamily  echo \$des-\/\texorpdfstring{$>$}{>}encrypt(substr(\$plaintext, 0, 8)); echo \$des-\/\texorpdfstring{$>$}{>}encrypt(substr(\$plaintext, 8, 8)); } {\ttfamily  echo \$des-\/\texorpdfstring{$>$}{>}encrypt(\$plaintext); }

The solution is to enable the continuous buffer. Although this will resolve the above discrepancy, it creates another, as demonstrated with the following\+:

{\ttfamily  \$des-\/\texorpdfstring{$>$}{>}encrypt(substr(\$plaintext, 0, 8)); echo \$des-\/\texorpdfstring{$>$}{>}decrypt(\$des-\/\texorpdfstring{$>$}{>}encrypt(substr(\$plaintext, 8, 8))); } {\ttfamily  echo \$des-\/\texorpdfstring{$>$}{>}decrypt(\$des-\/\texorpdfstring{$>$}{>}encrypt(substr(\$plaintext, 8, 8))); }

With the continuous buffer disabled, these would yield the same output. With it enabled, they yield different outputs. The reason is due to the fact that the initialization vector\textquotesingle{}s change after every encryption / decryption round when the continuous buffer is enabled. When it\textquotesingle{}s disabled, they remain constant.

Put another way, when the continuous buffer is enabled, the state of the \textbackslash{}phpseclib3\textbackslash{}\+Crypt\textbackslash{}\+DES() object changes after each encryption / decryption round, whereas otherwise, it\textquotesingle{}d remain constant. For this reason, it\textquotesingle{}s recommended that continuous buffers not be used. They do offer better security and are, in fact, sometimes required (SSH uses them), however, they are also less intuitive and more likely to cause you problems.
