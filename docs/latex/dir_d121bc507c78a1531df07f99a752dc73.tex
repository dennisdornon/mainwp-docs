\doxysubsection{Detailed Description}
\href{https://github.com/paragonie/constant_time_encoding/actions}{\texttt{ }} \href{https://packagist.org/packages/paragonie/constant_time_encoding}{\texttt{ }} \href{https://packagist.org/packages/paragonie/constant_time_encoding}{\texttt{ }} \href{https://packagist.org/packages/paragonie/constant_time_encoding}{\texttt{ }} \href{https://packagist.org/packages/paragonie/constant_time_encoding}{\texttt{ }}

Based on the \href{https://github.com/Sc00bz/ConstTimeEncoding}{\texttt{ constant-\/time base64 implementation made by Steve "{}\+Sc00bz"{} Thomas}}, this library aims to offer character encoding functions that do not leak information about what you are encoding/decoding via processor cache misses. Further reading on \href{http://blog.ircmaxell.com/2014/11/its-all-about-time.html}{\texttt{ cache-\/timing attacks}}.

Our fork offers the following enhancements\+:


\begin{DoxyItemize}
\item {\ttfamily mbstring.\+func\+\_\+overload} resistance
\item Unit tests
\item Composer-\/ and Packagist-\/ready
\item Base16 encoding
\item Base32 encoding
\item Uses {\ttfamily pack()} and {\ttfamily unpack()} instead of {\ttfamily chr()} and {\ttfamily ord()}
\end{DoxyItemize}\hypertarget{README.md_autotoc_md22}{}\doxysubsection{\texorpdfstring{PHP Version Requirements}{PHP Version Requirements}}\label{README.md_autotoc_md22}
