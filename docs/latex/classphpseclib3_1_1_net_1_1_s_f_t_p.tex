phpseclib3\textbackslash{}\+Net\textbackslash{}\+SFTP\+::put (\begin{DoxyParamCaption}\item[{}]{\$remote\+\_\+file}{, }\item[{}]{\$data}{, }\item[{}]{\$mode}{ = {\ttfamily self\+:\+:SOURCE\+\_\+STRING}, }\item[{}]{\$start}{ = {\ttfamily -\/1}, }\item[{}]{\$local\+\_\+start}{ = {\ttfamily -\/1}, }\item[{}]{\$progress\+Callback}{ = {\ttfamily null}}\end{DoxyParamCaption})}

Uploads a file to the SFTP server.

By default, \textbackslash{}phpseclib3\textbackslash{}\+Net\textbackslash{}\+SFTP\+::put() does not read from the local filesystem. \$data is dumped directly into \$remote\+\_\+file. So, for example, if you set \$data to \textquotesingle{}filename.\+ext\textquotesingle{} and then do \textbackslash{}phpseclib3\textbackslash{}\+Net\textbackslash{}\+SFTP\+::get(), you will get a file, twelve bytes long, containing \textquotesingle{}filename.\+ext\textquotesingle{} as its contents.

Setting \$mode to self\+::\+SOURCE\+\_\+\+LOCAL\+\_\+\+FILE will change the above behavior. With self\+::\+SOURCE\+\_\+\+LOCAL\+\_\+\+FILE, \$remote\+\_\+file will contain as many bytes as filename.\+ext does on your local filesystem. If your filename.\+ext is 1MB then that is how large \$remote\+\_\+file will be, as well.

Setting \$mode to self\+::\+SOURCE\+\_\+\+CALLBACK will use \$data as callback function, which gets only one parameter -- number of bytes to return, and returns a string if there is some data or null if there is no more data

If \$data is a resource then it\textquotesingle{}ll be used as a resource instead.

Currently, only binary mode is supported. As such, if the line endings need to be adjusted, you will need to take care of that, yourself.

\$mode can take an additional two parameters -\/ self\+::\+RESUME and self\+::\+RESUME\+\_\+\+START. These are bitwise AND\textquotesingle{}d with \$mode. So if you want to resume upload of a 300mb file on the local file system you\textquotesingle{}d set \$mode to the following\+:

self\+::\+SOURCE\+\_\+\+LOCAL\+\_\+\+FILE \texorpdfstring{$\vert$}{|} self\+::\+RESUME

If you wanted to simply append the full contents of a local file to the full contents of a remote file you\textquotesingle{}d replace self\+::\+RESUME with self\+::\+RESUME\+\_\+\+START.

If \$mode \& (self\+::\+RESUME \texorpdfstring{$\vert$}{|} self\+::\+RESUME\+\_\+\+START) then self\+::\+RESUME\+\_\+\+START will be assumed.

\$start and \$local\+\_\+start give you more fine grained control over this process and take precident over self\+::\+RESUME when they\textquotesingle{}re non-\/negative. ie. \$start could let you write at the end of a file (like self\+::\+RESUME) or in the middle of one. \$local\+\_\+start could let you start your reading from the end of a file (like self\+::\+RESUME\+\_\+\+START) or in the middle of one.

Setting \$local\+\_\+start to \texorpdfstring{$>$}{>} 0 or \$mode \texorpdfstring{$\vert$}{|} self\+::\+RESUME\+\_\+\+START doesn\textquotesingle{}t do anything unless \$mode \texorpdfstring{$\vert$}{|} self\+::\+SOURCE\+\_\+\+LOCAL\+\_\+\+FILE.
